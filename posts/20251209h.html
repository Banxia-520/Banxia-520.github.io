XSS 跨站脚本攻击

1. 什么是 XSS

XSS（Cross-Site Scripting，跨站脚本攻击）是指攻击者将恶意脚本注入到网页中，当其他用户访问该页面时，脚本在用户的浏览器端执行。

为避免与 CSS（层叠样式表）重名，故命名为 XSS。

XSS 的危害

危害 说明
窃取 Cookie 盗取用户 Session，冒充身份登录
钓鱼攻击 伪造登录表单，窃取账号密码
键盘记录 记录用户输入，获取敏感信息
页面篡改 修改页面内容、插入恶意广告
挖矿脚本 在用户浏览器中运行加密货币挖矿脚本
配合 CSRF 发起自动化的跨站请求
传播蠕虫 自我复制的 XSS 蠕虫（如 2005 年 MySpace 蠕虫事件）

2. XSS 三种类型

2.1 反射型 XSS（Non-Persistent）

特点：非持久化，恶意代码在 URL 参数中，需要诱导用户点击。

攻击流程：

常见场景： 搜索框、错误提示、URL 参数输出。

2.2 存储型 XSS（Persistent）

特点：持久化，恶意代码被存储在服务器上（数据库、文件系统等），任何访问该页面的用户都会中招。

攻击流程：

常见场景： 评论区、个人资料、留言板、富文本编辑器、帖子内容。

2.3 DOM 型 XSS

特点：恶意代码不经过服务器，完全在客户端通过 JavaScript 操作 DOM 时触发。

常见场景： 前端框架未安全处理 location.hash、document.URL、window.name 等。

三种类型的对比

维度 反射型 存储型 DOM 型
持久化 否 是 否
代码位置 URL 参数 服务器存储 URL Hash / 客户端
触发方式 用户点击链接 访问正常页面 正常页面交互
危害范围 特定受害者 所有访问者 特定受害者

5. XSS 漏洞防御

方案一：输出编码（核心）

将用户输入的数据在输出到 HTML 前进行转义：

上下文 转义方式 示例
HTML 标签体内 HTML 实体编码 &lt;script&gt;
HTML 属性中 属性值转义 + 引号包裹 onclick="..."
JavaScript 字符串 Unicode 转义 \u003cscript\u003e
CSS 中 尽量不使用用户输入 -
URL 中 URL 编码 %3Cscript%3E

总结

维度 说明
核心原因 用户输入被当作代码执行，而非数据处理
三种类型 反射型（URL）、存储型（数据库）、DOM 型（前端）
利用方式 窃取 Cookie、页面篡改、键盘记录、钓鱼、CSRF
根本防御 输出编码 + CSP + HttpOnly 三层防护
一句话总结 永远不要相信用户输入，输出时必须转义
