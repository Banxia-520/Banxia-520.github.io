CSRF 跨站请求伪造

1. 什么是 CSRF

CSRF（Cross-Site Request Forgery，跨站请求伪造），又称 XSRF，是一种攻击者诱导用户在已登录的站点上执行非自愿操作的攻击方式。

核心思想

攻击者伪造请求，利用受害者已登录的身份（Cookie、Session），冒充受害者向目标服务器发送恶意请求，完成转账、改密、发帖等操作。

与 XSS 的区别

攻击类型 攻击对象 信任基础 攻击方式
XSS 用户 信任站点内容 向页面注入恶意脚本
CSRF 站点 信任用户请求 伪造用户身份发送请求

XSS 利用了用户对网站的信任，而 CSRF 利用了网站对用户浏览器身份凭证的信任。

2. CSRF 攻击原理

CSRF 攻击需满足三个条件：

  1. 受害者已登录目标站点，且 Session/Cookie 尚未过期
  2. 目标站点未对敏感操作做来源校验（如无 Token、无 Referer 检查）
  3. 攻击者能够诱导受害者发起请求（如通过钓鱼链接、恶意图片等）

3. CSRF 漏洞检测

黑盒测试（手工）

  • 对目标站点的敏感操作（增删改、转账、改密等）拦截请求
  • 去掉 Referer 头或修改 Origin 字段后重放，观察是否被拦截
  • 使用 Burp Suite 生成 CSRF POC，看是否能成功触发

白盒审计（代码审查）

检查敏感接口是否存在以下防御措施：

  • Token 校验：请求是否携带了随机的 anti-CSRF Token，且服务端验证
  • Referer/Origin 校验：是否验证了请求来源
  • SameSite Cookie：是否设置了 SameSite=Strict/Lax
  • 二次确认：敏感操作是否有验证码或确认步骤

4. CSRF POC 生成与利用

使用 Burp Suite 生成 POC

  1. 拦截目标请求（GET 或 POST）
  2. 右键 → Engagement Tools → Generate CSRF PoC
  3. Burp 自动生成包含请求参数和表单自动提交的 HTML 页面
  4. 将该 HTML 放置于攻击者服务器上，诱导受害者访问

利用方式

  • 配合 XSS：通过 XSS 注入自动触发 CSRF 请求
  • 钓鱼页面：伪装成正常页面，在后台发送 CSRF 请求
  • 图片/资源劫持：在第三方论坛、评论区嵌入恶意资源链接

现代浏览器已逐步增强 CSRF 防御机制，但服务端的 Token 校验仍是根本解决方案。

6. CSRF 防御措施

措施 说明 强度
CSRF Token 每个表单/请求生成随机 Token，服务端验证 高
SameSite Cookie 设置为 Strict 或 Lax 高
Referer/Origin 校验 验证请求来源是否合法 中
验证码 敏感操作要求输入验证码 高
二次确认 修改关键信息需再次输入密码 中

SSRF 服务端请求伪造

1. 什么是 SSRF

SSRF（Server-Side Request Forgery，服务端请求伪造）是指攻击者利用服务器提供的对外请求功能，诱使服务器向内部网络或其他受保护资源发起请求的漏洞。

与 CSRF 的本质区别

CSRF SSRF
请求发起方 客户端（浏览器） 服务端
攻击目标 以用户身份操作 访问内网/隐藏资源
利用方式 伪造用户请求 伪造服务器请求

SSRF 的核心危害

  • 扫描内网端口、探测内网资产
  • 读取服务器本地文件（file:// 协议）
  • 攻击内网服务（如 Redis、MySQL、Elasticsearch）
  • 突破网络边界隔离

2. SSRF 漏洞挖掘

容易产生 SSRF 的业务场景

业务场景 功能描述
社交分享 获取分享链接的标题、缩略图、描述
在线翻译 翻译指定 URL 的网页内容
图片加载/下载 通过 URL 加载或下载远程图片到本地
转码服务 将网页内容转为适合手机浏览的格式
网站采集/抓取 采集指定 URL 的内容或元数据
云服务 远程检测网站是否存活、健康检查
邮件系统 接收/发送邮件时解析外部链接
文档处理 解析文档中的远程资源引用
数据库功能 如 MongoDB 的 copyDatabase、MSSQL 的 xp_cmdtree

5. SSRF 漏洞防御

防御措施 实现方式
URL 白名单 只允许请求预设的可信域名/IP
协议白名单 仅允许 http:// 和 https://，禁用 file://、gopher:// 等协议
内网 IP 黑名单 过滤 127.0.0.1、10.x.x.x、172.16~31.x.x、192.168.x.x
端口限制 只允许 80、443、8080 等常用端口
响应校验 验证返回数据格式是否符合预期
统一错误信息 避免通过错误信息推测内网端口状态
禁用重定向跟随 或者对重定向后的 URL 再次进行校验

总结对比

维度 CSRF SSRF
攻击发起方 客户端（浏览器） 服务端
信任基础 Cookie/Session 身份 服务器网络权限
主要危害 冒充用户操作 访问内网/读取本地文件
防御核心 Token + SameSite Cookie URL/协议/IP 白名单校验
常见场景 转账、改密、发帖 图片抓取、翻译、采集

一句话总结：CSRF 是借用户之手攻击站点，SSRF 是借服务器之手攻击内网。
