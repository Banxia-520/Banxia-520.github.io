XXE 外部实体注入漏洞

1. 什么是 XXE

XXE（XML External Entity Injection，XML 外部实体注入）是指应用程序在解析 XML 输入时，未对外部实体引用做任何安全限制，导致攻击者可以注入恶意外部实体，从而引发文件读取、SSRF、命令执行等危害。

前置知识：XML 实体机制

XML 实体类似于编程语言中的宏或变量，用于定义可复用的数据片段：

XXE 漏洞正是利用了外部实体这一特性，让服务器加载攻击者指定的资源。

2. XXE 漏洞原理

产生根源

应用程序使用 XML 解析器处理用户可控的 XML 数据时，默认开启了外部实体解析功能，且未对实体来源做任何过滤。

总结

维度 说明
漏洞本质 XML 解析器加载了攻击者控制的外部实体
攻击方式 有回显直接读 / 无回显 OOB 外带 / SSRF 内网探测
利用协议 file:// 读文件、http:// SSRF、php:// 伪协议
核心防御 libxml_disable_entity_loader(true) 或等效配置
一句话总结 只要 XML 解析器允许加载外部资源，且来源可控，就存在 XXE

补充：不要认为”JSON 接口就不会有 XXE”——如果框架同时支持多种 Content-Type 解析，改请求头即可绕过。
