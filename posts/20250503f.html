PHP 反序列化漏洞

1. 什么是反序列化漏洞

前置概念：序列化与反序列化

序列化是将对象转换为可存储或传输的字符串（如 JSON、二进制）的过程，反序列化则是将字符串恢复为对象的过程。

漏洞本质

反序列化漏洞是指应用程序对用户可控的反序列化输入没有做任何校验，攻击者通过构造恶意序列化数据，在反序列化过程中触发对象的魔术方法，进而执行恶意代码。

2. PHP 魔术方法

反序列化漏洞的核心是利用 PHP 的魔术方法（Magic Methods），它们在特定时机自动调用：

魔术方法 触发时机 说明
__construct() 对象被创建时 构造函数
__destruct() 对象被销毁时 最常用的攻击入口
__wakeup() unserialize() 调用时 反序列化直接触发
__sleep() serialize() 调用时 序列化时触发
__toString() 对象被当作字符串时 如 echo $obj
__call() 调用不可访问的方法时 $obj->nonexist()
__get() 读取不可访问的属性时 $obj->undefinedProp
__set() 给不可访问的属性赋值时 $obj->prop = value
__invoke() 对象被当作函数调用时 $obj($args)

3. POP 链构造

6. 反序列化漏洞防御

防御措施 说明 优先级
不使用 unserialize() 改用 JSON 等安全格式 最高
严格白名单 PHP 7.0+ 支持 allowed_classes 参数 高
哈希校验 序列化数据附加签名，防止篡改 高
输入加密 使攻击者无法构造有效数据 中

总结

维度 说明
核心原因 unserialize() 接收用户可控输入且未做限制
利用手法 构造恶意序列化数据 → 触发魔术方法 → POP 链 → RCE
关键角色 __destruct、__wakeup、__toString 三个魔术方法
绕过技术 __wakeup 绕过、引用绕过、字符逃逸
根本防御 用 JSON 替代序列化，或使用 allowed_classes 白名单
一句话总结 反序列化的本质是”攻击者决定了对象的属性和方法，从而控制程序执行流”
