文件包含漏洞

1. 什么是文件包含漏洞

文件包含漏洞（File Inclusion）是指应用程序通过动态引入文件时，未对用户输入的路径做充分过滤，导致攻击者可以包含恶意或非预期文件。

两种类型

类型 定义 危害程度
LFI（本地文件包含） 包含服务器本地的文件 中高
RFI（远程文件包含） 包含远程服务器上的文件 极高

RFI 默认在 PHP 5.2 之后需开启 allow_url_include 才可用，实际场景较少，但仍存在于部分遗留系统中。

常见危险函数

函数 作用 是否支持 RFI
include() 包含并执行文件，失败时产生警告 是
include_once() 同 include，但只包含一次 是
require() 包含并执行文件，失败时产生致命错误 是
require_once() 同 require，但只包含一次 是
highlight_file() 高亮显示文件内容 否
file_get_contents() 读取文件内容 是
readfile() 读取并输出文件内容 是

4. 文件包含漏洞防御

防御措施 说明
禁止动态文件包含 使用白名单映射，而非直接拼接用户输入
关闭危险配置 allow_url_include = Off、allow_url_fopen = Off
路径过滤 禁止 ../、..\\ 等目录遍历字符
禁用伪协议 确认不需要的协议（如 php://input）可禁用
文件权限最小化 Web 用户不可读敏感文件

安全示例：白名单映射

核心原则：让用户选择”要哪个”，而非让用户决定”路径是什么”。

总结

维度 说明
核心问题 用户输入直接被拼接到文件包含路径中
LFI 常用技巧 目录遍历、伪协议、日志注入、phpinfo 竞争
RFI 条件 allow_url_include = On
根本防御 白名单映射 + 关闭 allow_url_include
一句话总结 文件包含的本质是路径可控 + 代码执行，用白名单替代路径拼接即可根除
